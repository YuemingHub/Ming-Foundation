# REF-0033 — Kernel Core Review and Evidence Protocol

## 1. Current boundary

`PreparedNotExecuted`: zero real reviewers, human-use approvals, live evidence
environments, affected-person sessions or implementation assessments.

## 2. Review classes

Architecture/interoperability; life ethics/agency; affected-person and child
rights; privacy/consent/deletion; safety/handoff/incident/appeal;
professional/domain; accessibility/language; commercial/public claims;
legal/jurisdiction; implementation/failure; security/evidence; translation.

## 3. Source and MOS review

For every KCR, verify exact source metadata and locator, source role,
transformation type, no strengthening/weakening, implementation-neutral
verification and evidence privacy. KERNEL-0000/0001 must substantively include
all MOS-required sections.

## 4. Evidence classes

Repository structure, exact source traceability, synthetic scenario, reference
implementation, implementation trace, affected-person review, professional
review, incident/failure, longitudinal outcome and public-claim audit.

Repository validation is structural evidence only.

## 5. Completion gate

Completion requires real qualified participation, exact versions, positive and
minority findings, Critical/High disposition, approved evidence handling,
synchronized Markdown/JSON and accountable decision. Round 07 authorizes no
recruitment, human sessions, CP2 or CP3.
