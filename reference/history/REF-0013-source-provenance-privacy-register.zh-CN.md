# REF-0013 — MingOS 历史来源、追溯与隐私登记

## 1. 目的

历史来源用于回答“这个概念从哪里来、后来如何变化”，不是为了把全部私人
对话、家庭案例和实现细节迁入公共仓库。

## 2. 来源等级

| 来源类型 | 默认用途 | 不得用于 |
|---|---|---|
| 当前公开仓库 | 确认当前权威、状态和依赖 | 推断未写入仓库的历史原文 |
| 私人历史文件 | 确认概念出现、语境和演化 | 直接公开私人表达或家庭信息 |
| 实现资料 | 理解产品对象和运行历史 | 反向定义 Foundation 与 Charter |
| 实践资料 | 理解方法来源和真实场景 | 将单案例升级为普遍规律 |
| 二手摘要 | 帮助定位待查来源 | 作为逐条原文、科学证据或治理决定 |
| 外部文献 | 未来科学和伦理审查 | 仅凭名称为 MingOS 背书 |

## 4. 隐私与公开规则

不得进入公共来源登记的内容：

- 姓名、联系方式、学校、地点和可识别家庭结构；
- 未获授权的咨询、聊天和语音原文；
- 儿童的私人表达；
- 医疗、心理、法律和安全敏感细节；
- 可以通过组合重新识别当事人的案例线索；
- 仅因“对方法很有价值”而采集的脆弱经历。

允许进入公共仓库的形式：

- 去身份化概念；
- 方法结构；
- 不含个案内容的历史时间线；
- 来源类别；
- 对隐私处理的说明；
- 无法恢复原文的明确登记。

## 5. 引用强度

### 高可信度

- 当前仓库具体提交中的完整文件；
- 经过校验的公开决议和规范；
- 可以逐字定位的历史文件。

### 中可信度

- 完整历史文件但未进入治理；
- 多份来源一致支持的概念作用；
- 实现文档与代码互相印证。

### 低可信度

- 对话摘要；
- 个人记忆；
- 没有原文的集合名称；
- 只有后续文本可反推的旧概念。

低可信度来源只能产生“待核对”记录，不能产生历史事实或状态提升。

## 6. 九条公理与十六条禁止的特殊处理

目前只能公开确认：

- 两个集合名称存在；
- 它们曾位于早期底座；
- 分别承担正向公理与负面禁止的角色；
- 后续文件吸收了相近方向。

目前不能公开确认逐条原文。

如果未来找到原始材料，应：

1. 先做隐私与完整性审查；
2. 计算文件哈希；
3. 记录来源时间和版本；
4. 与当前文件逐条映射；
5. 不因“历史更早”而自动恢复权威；
6. 通过独立 PR 决定是否建立受限历史附录。

## 7. 家庭实践资料处理

家庭实践是 MingOS 知识的重要来源，但家庭不是数据供应商。

方法可以从案例中形成，公开文本必须做到：

- 不复述可识别事件；
- 不把一方叙述当作完整家庭真相；
- 不以“教学案例”取消同意与撤回；
- 不把成功故事作为方法有效性的唯一证据；
- 保存失败、副作用和当事人异议；
- 区分实践启发与科学结论。
